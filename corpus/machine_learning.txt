Lecture 3
	•	Parametric model: assume that the data follows a certain distribution 
	•	Number of parameters (degrees of freedom) independent of data
	•	Fixed number of parameters
	•	Non-parametric model: can’t know how distribution will look like
	•	Degree of freedom increase with more data

	•	In supervised learning, you build a model on training data and then make prediction on new and unseen data 
	•	Accurate predictions mean you were able to generalize from training to test set
	•	A model too simple causes underfitting as you don’t capture all features
	•	A model too complex causes overfitting as it works too well on the training set but can’t generalize on new data 
	•	The more features you use to build a model the better you explain the actual behavior of one individual point but you’re sacrificing the ability to generalize 
	•	Therefore, you should find the “sweet spot”
	•	** trade-off between complexity and generalization

	•	Complexity is tied to variation; the larger the variation the more complex your model can be without overfitting
	•	Larger datasets allow building complex models
	•	Bias is an error caused by approximating a real-world problem with a simplified one. 
high bias  strong assumptions  simple model  underfitting 
	•	Variation: error caused by the model’s sensitivity to small fluctuations in the training data sets 
High variance  high flexibility/sensitivity  complex model  overfitting
	•	High variance means low bias, vice versa *not always hand in hand 

	•	K-nearest neighbors: to make a prediction for new data point by finding the closest data points in the training dataset
	•	The simplest version only takes 1 NN which is the closest training point to the point we want to make a prediction for 
	•	Few neighbors  high model complexity, vice versa
	•	More neighbors or features determines time complexity
	•	Extreme case is that the number of neighbors = number of data points in training set so each test point would have the same neighbors, and the predictions will all be the same; class that’s more frequent 
	•	Advantage: easy to understand, gives reasonable performance without a lot of adjustments, reasonably fast
	•	Drawbacks: slow with large datasets, can’t handle many features 

	•	Nearest centroid algorithm finds the center of gravity of all the data points of one class. So can draw the decision boundary based on these centroids and check which centroid the new data point is closest to
	•	Allows to calculate it once for all coordinates rather than every data point and don’t use all the data anymore
	•	Nearest shrunken centroid shrinks each class centroid towards the overall centroid to find the important features that help distinguish between classes or clusters 
	•	How? (Extra) calculates the distance between class centroid and general centroid. The distances tell us how much a feature helps distinguish between classes, so a large distance says that it plays an important role. Any feature with a small distance is shrunk towards the overall centroid, dampening its effect. In the end, only features with large distances remain distinct from overall centroid

	•	When finding the right parameter setting, we are running the analysis many times so you’re reusing the test data and picking the best model on the test set and so can’t assess if the model is good, leading to overfitting 
	•	3-fold split:
	•	a validation set that’s taken out of the training set to build the model and find the right parameter setting
	•	The test set now validates the model’s performance and check if it is true 
	•	Pro: fast and simple
	•	Con: high variance, bad use of data

	•	Cross validation: data is split repeatedly, and multiple models are trained 
	•	Common version is k-fold cross validation; usually 5-10
	•	Data is split into k parts (folds) then the model is trained on k-1 folds and tested on remaining fold, each time using a different fold. Take the average result to find cross validation accuracy

	•	Pro: more stable and more data, multiple splits give information on sensitivity,  
	•	Con: k times slower
	•	High variance in accuracy between folds could imply that the model is very dependent on folds in training but could be a consequence of small dataset
	•	** cross-validation doesn’t build a model it evaluates how well a given algorithm will generalize when trained on a specific dataset

	•	Splitting data set in k folds starting with first 1/kth of data set might not always work so you should use stratified k-fold cross validation in which you split data such that proportions between classes are the same in each fold as the dataset 
	•	leave one out cross validation: #of folds=#data points, using a single point as test set, slow and high variance
	•	repeated Kfold: apply Kfold or stratified fold multiple times with shuffled data; lower variance
	•	shuffle split cross validation: randomly shuffle and split. It is flexible but no guarantee that points are tested once
	•	hard to keep track of shuffling and use some data multiple times
	•	nested cross validation
	•	outer loop: for each group train the model with the optimal parameters from the inner loop and then estimate of generalization performance across test folds
	•	within each fold of the outer loop, the inner loop is used to perform hyperparameter tuning, evaluate performance on validation fold and then choose the hyperparameters of the best model
	•	cross validation with groups (Group KFold): data is split such that samples belonging to the same group are either all in the training set or all in the test set
	•	time series cross validation: first split data in training and testing then for each subsequent split expand training set to include more recent data 
	•	example: split 1: train jan-feb, test march, split 2: train jan-march, text April, etc


	•	must tune parameters to improve model’s generalization performance 
	•	to find values of the important parameters of a model you can use grid search 
	•	grid search: used for hyperparameter tuning; it exhaustively tries every combination of hyperparameters provided to find the best model
	•	hyperparameters are set before training and are tuned manually while parameters are learned from data during training and are optimized automatically
Lecture 4
	•	linear models for regression are regression models for which the prediction is a line for a feature, plane for two, hyperplane for higher dimensions
	•	if you have more features than training data, you can perfectly fit a linear model into training set
	•	 so target value is a LC of features
	•	Weights or coefficients (w) are slope parameters
	•	The offset it the intercept b

	•	In normal regression you would integrate the equation but now you have vectors because we have many features
	•	Simplest linear method is ordinary least squares which finds parameters w and b that minimizes MSE between prediction and true regression targets
 coefficients estimates rely on independence of features
	•	if you have data set where # rows < # features it tells us ability what can be learned from data set; like DNA

	•	Linear regression has no parameters so you can’t control model complexity
	•	If R^2 for training is much greater than test set  overfitting  need to control complexity
	•	R^2 (coefficient of determination) how well a model fits
	•	So use ridge regression

	•	The coefficients ‘w’ are chosen so that they predict well on training and to fit an additional constraint; their magnitudes must be as small as possible so that each feature has little effect on the outcome as possible and aren’t that predictive
	•	The constraint is an example of regularization: restricting model to avoid overfitting by dampening the effect of features
	•	Ridge uses L2 regularization 
	•	Always has a unique solution
	•	Has tuning parameter alpha that controls how strongly coefficients are pushed to 0
	•	If alpha is too low, we remove the effect of regularization and end up overfitting
	•	Ridge model makes a tradeoff between simplicity of model and performance on training set
	•	The more data is available the more linear regression will perform like ridge model as it becomes harder for a model to overfit or memorize data 

	•	In general: regularized empirical risk minimization
	•	used for any model while ridge is a specific example of ERM for linear models with L2 regularization 
	•	so L2 norm means that R(F) = ||w||^2
 
	•	Optimization function in ML is defined by:
	•	the loss factor: general function 
	•	regularization term

	•	Can add/remove features to make predefined choices before making the model
	•	Can also have linear combinations of features 

	•	Can find the best parameter setting for your alpha score just like you would in KNN
	•	Increasing alpha forces coefficients to move closer to 0, which decreases training set performance but might help with generalization

	•	Lasso regression: same as ridge but uses L1 regularization. As a result, come coefficients are exactly 0 and some features can be ignored

	•	Makes the model easier to understand and simpler as you are using less features 
	•	It equally dampens every feature rather than dampen coefficients much more when they have larger magnitudes 



	•	Elastic net: combines ridge and lasso so you have 2 parameters to tune

	•	In regression, not all data perfectly resonates with linear models, and may contain outliers
	•	Either remove it since it’s not representative
	•	Use majority and just ‘ignore’ it 
	•	However, outliers affect our model so use RANSAC algorithm 
	•	First select minimum amount of data required to fit a model
	•	fit a model
	•	RANSAC classifies each point as inlier or outlier based on residual (distance between actual and prediction). Inlier if residual is below residual threshold
	•	Choose model with highest inliers, if they have same number of inlier use lowest residual so better score
	•	Huber loss
	•	Different from ridge as it applies a linear loss to samples classifies as outliers
	•	It doesn’t ignore outliers but reduce their effect
**
- In linear regression you can’t control complexity 
- To reduce overfitting, you use ridge regression where you have a tuning factor alpha that push coefficient ‘w’ closer to 0 to reduce the effect of features 
- In ridge model there’s a tradeoff between simplicity and performance on training set
- The bigger the data set the smaller the effect of regularization
- Lasso regression is the same as ridge but uses L1 regularization instead of L2 so some coefficients might be 0
- Lasso regression equally dampens all features
- Elastic net: combines ridge and lasso
- use RANSAC algorithm to find a model for data with outliers

	•	Scaling data might help:
	•	dampen effect of feature based on their scale and prevents dominance from large-scale features, allowing the model to learn their true importance based on their relationship to the target variable rather than their scale.
	•	reduce computational expenses since the larger the scale the larger the range
	•	Sometimes where the point resides is not that important only relationship or patterns do
	•	All models using a distance measure will be affected by order of magnitude of the features and usually benefit from scaling data 

	•	Box plot can tell you where most of the data is found 
	•	Influence of outlier is not big so can ignore large outliers 
	•	scaling doesn’t change the visual structure of data but it changes classification
	•	might catch patterns more effectively because u don’t emphasize on features due to their order of magnitude

	•	which type of scaling depends on features 
	•	StandardScaler scales the data such that the mean is 0 and the variance is 1
	•	All features have same magnitude
	•	Doesn’t ensure min and max values for features 
	•	RobustScaler uses median and quartiles so ignores outlier 
	•	MinMax scaler shifts data such that all features are between 0 and 1
	•	Normalizer scales each data point such that the feature vector has Euclidean length of 1
	•	Projects data points on unit circle 
	•	Use when only direction of data points matters not their magnitudes. E.g.: finding direction of shopper like frequent vs occasional buyer instead of how much they spend 
	•	Box-cox scaling method to take away skewness in data to apply linear regression models
	•	Because sometimes LR models can’t find patterns if data is not linearly distributed
	•	Periodic scaling: periodic patterns in data like cosine and sine e.g.: selling iice-cream in summer and winter

	•	When scaling you only scale using training data.
	•	Must apply the same transformation for training and test set. So always subtracts the training set min and divides by training set range to maintain the same structure
	•	So you transform the training and test data by the same scaling

	•	Sparse data:
	•	Data with many zeros – only store non-zero entries
	•	Subtracting anything will make the data “dense” as it changes 0 values to non-zero values and blow the RAM 
	•	Only scale don’t center to maintain the zero entries
	•	E.g.: in recommender systems, users don’t always check all the products in the store so you will have a lot of 0 entries

	•	Categorical variables: discrete features, not usually numeric, and cant be handles by default by a regression algorithm 
	•	To represent categorical variables using the one-hot-encoding known as dummy variables 
	•	Dummy variables replace a categorical variable with one or more new features (new columns) with values 0 and 1; so 1 if yes 0 if no
	•	Transforms all possible options for the category by creating new columns 
	•	Remove original column 
	•	If there are no fixed categories like writing male or man, or different spelling etc you might need to preprocess the data. Could do so by using value_counts

	•	Coding:
	•	Do not use get_dummies separately om train and test sets
	•	Mismatch of columns and training and test sets will have different number of features. E.g.: when you have A&B in your training data set but only A in your test, 2 dummy variables will be created in training and 1 in test

	•	To avoid it:
	•	combine training and test sets temporarily before encoding 
	•	align column names after encoding 
	•	explicitly provide categories 
	•	Sometimes we don’t know if the data is already categorical or not

	•	For columns: might lose an informative feature and discard critical information
	•	For rows: sometimes you don’t have a choice for dropping a sample and are forced to make a prediction, introducing bias. But sometimes you don’t have a way to deal with missing values 
	•	Imputation methods: fills missing values usually unsupervised and only make use of information of features on the training data 
	•	Estimates missing values 
	•	Could fill with mean or median
	•	Disadvantage:
	•	Data moves to the center and distributions are harder to distinguish as there are many identical values

	•	Find k nearest neighbors that have non-missing values
	•	Replace using average of neighbors 
	•	Preserves more of the original data distribution 
	•	Disadvantage: additional hyper-parameter and large time complexity 
	•	Model driven imputation: 
	•	Train regression model for missing values 
	•	Iterate over columns and create new model
	•	Retrain after filling in 
	•	Takes the longest

	•	Making model less complex to avoid overfitting

	•	Less storage for model and dataset
	•	Less complex so you can interpret the model better

	•	Unsupervised feature selection: discard some features based on statistics
	•	Remove features that are mostly constant or have variance 0. Note that it doesn’t mean anything if variance is small since you can just rescale it 
	•	Remove correlated features as they explain the same thing and make same predictions. However, sometimes they might be important like temperature and humidity (both predict weather).
	•	Pro: doesn’t need target variable and is quick 
	•	Con: discards values that might be important for predictions and removing low-variance or correlated features might eliminate critical information

	•	Reduces redundancy but doesn’t guarantee that the discarded information isn’t important for you prediction problem 

	•	Univariate vs multivariate: how many features you analyze at once to understand if feature should be kept 
	•	Univariate: tests if each feature is significantly related to the target variable; each feature is evaluated separately without considering how it interacts with other features 
	•	For classification problems, this is often done using ANOVA
	•	Features that are important when combined with another would be discarded
	•	Fast but is unrelated to the specific model you plan to use
	•	Simplest way
	•	Check p-values; low p-value means that the relationship between feature and target variable is significant
	•	Depending on if its classification or regression, can add t-test, f-test or chi-squared test and select features highly related to target
	•	Mutual information: amount of information two variables shares; contains difference in information gained or loss by including a feature
	•	If they’re independent, it’ll be 0
	•	Doesn’t assume linear model like f statistics 

	•	Model based or not: choose features from outcome of model or internal learnings like coefficients

	•	Finds subset of features in which model performs best

	•	if features are correlated lasso pick one feature at random since the information is already captured by one of them

	•	Iterative feature selection uses a series of models with varying numbers of features 
	•	Start with single feature, find most important feature, add, iterate; adds each feature one by one
	•	Fit model, find least important feature, remove, iterate; remove one by one 
	•	Since a series of models are build its computationally expensive

Lecture 8
	•	Trees don’t care about the scale of the data, so you don’t have to do a lot of preprocessing 
	•	Also doesn’t care about the distribution of your data, are interpretable and can work with missing data 
	•	Binary classification trees are binary tree where each node corresponds to a question asked about the data
	•	For continuous features: “questions” are thresholds on single features 
	•	For each split: exhaustive search over all features and thresholds to find the best and minimize impurity
	•	Impurity = 0: perfect segregation of data
	•	Nodes correspond to a split and the leaves correspond to regions 
	•	Learning a decision tree means learning the sequence of if/else questions that gets us to the true answer most quickly.
	•	These questions are called tests

	•	For classification, gini-index and cross-entropy, are two criteria to preserve the purity of the leaf 
            


	•	Cross entropy is the entropy of the distribution of classes
	•	Minimizing cross entropy aligns the predicted distribution as closely as possible to the true distribution

	•	Traverse the tree based on feature tests 
	•	Predict the most common class in leaf 

	•	For regression trees: 
	•	Impurity criteria: MSE and MAE
	•	Usually predict the mean over the targets in a leaf
	•	Tendency to get deep as all targets are usually distinct float numbers
	•	Without regularization/pruning each leaf will contain a single point to be pure 
	•	Pure means that the tree is 100% accurate on training data -> overfitting 

	•	Parameter tuning:
	•	To restrict growth, you use pre-pruning and post-pruning
	•	Pre-pruning: restrict while growing; stop creation of tree early on
	•	post-pruning you build the whole tree (greedily) and then you merge notes; remove nodes that contain little information
	•	can limit tree size in pre-pruning using stretegies below to reduce overfitting
	•	max_depth: very fast as you don’t have to go deep but if you’re only building a single decision tree it’s not good
	•	max_leaf_nodes: prioritizes the greatest impurity first
	•	min_samples_split: require a minimum number of points in a node to keep splitting
	•	etc.
	•	DecisionTreeClassifier only implements pre-pruning
	•	Will continue to split the data until all leaves are pure so tree is 100 % accurate on training set and hence is overfitting (poor generalization)
	•	Used when you need to easily explain and interpret data to non-experts and when using small amount of data
	•	Completely invariant to scaling of data

	•	Recall measures how many of the positive samples are captured by the positive predictions

	•	Identify all positive samples; important to avoid false negatives

	•	F1-score: harmonic mean of precision and recall

Lecture 11
	•	Clustering is the task of partitioning the dataset into groups called clusters 
	•	unlike classification, clustering doesn’t rely on predefined labels
	•	no ground truth or target variable to learn from
	•	the algorithm only tells you if data points belong to the same cluster; labels are meaningless until examined
	•	briefly:
	•	group data
	•	identify groups by assigning cluster labels to each data point
	•	points within a cluster are similar
	•	points in different clusters are different
	•	number of clusters might be pre-defined
	•	notions of similar and different depend on algorithm or use specified metrics

	•	cluster algorithms have many parameters including number of clusters
	•	to pick right number of clusters you must have prior knowledge of the data or need to inspect results manually
	•	sometimes you don’t know

	•	Why clustering?
	•	Explore and understand structure of data
	•	Are there coherent groups? How many?
	•	Data partitioning: divide data by group before further processing
	•	Summarize or compress data by representing each group by center
	•	Unsupervised feature extraction: derive features from clusters or cluster distances

	•	K-means clustering:
	•	Pick number of clusters k 
	•	Pick k random points as cluster centers
	•	While cluster center changes, assign each data point to closest cluster center and recompute cluster centers as the mean of assigned points
	•	Always converges

	•	Objective function for k-means:
	•	Finds local minimum of minimizing squared distances (Exact solutions is NP hard)
	•	New data points can be assigned cluster membership based on existing clusters
	•	
	•	Sum of the distances of the cluster centers to all the points in the clusters

	•	You can assign cluster labels to new data points via the predict method in Kmeans
	•	initialization is random so running the algorithm multiple times can lead to different cluster numbers being assigned to the same group of data points
	•	the way points get allocated affects how clusters are formed
	•	each cluster is represented by its center and the boundary between clusters are straight lines so k-mean can only identify convex clusters
	•	k-means assumes that all clusters are roughly the same size and finds the boundaries by finding the midpoint so might split the clusters incorrectly
	•	even if right number of k is known, k-mean algorithm can’t recover them
	•	k-means performs poorly if clusters have weird shapes

	•	Advantages of k-means:
	•	Easy to understand and implement as each cluster is represented by a center (simple)
	•	Runs quickly 
	•	Scales easily to large datasets 
	•	Disadvantages:
	•	Relies on random initialization so outcome depends on random seed
	•	Restrictive assumptions made on the shape of clusters; clusters are always convex and simple
	•	Cluster boundaries equidistant to centers
	•	Requirement to specify the number of clusters you’re looking for
	•	Can’t model covariances well

	•	While PCA and NMF (learn later) tries to express data points as sum over some components, k-means tries to represent each data point using a cluster center 
	•	Vector quantization: viewing k-means as a decomposition method where each point is represented using a single component
	•	In vector quantization using k-means is that we can use many more clusters than input dimension
	•	Using PCA or NMF while reducing to one dimension destroys data structure

	•	Computational properties: 
	•	Naïve implementation: n_cluster * n_samples distance calculations per iteration
	•	Fast “exact” algorithms: Elkan’s, Ying-Yang, etc.
	•	Approximate algorithms: minibatch KMeans which is much faster

	•	Initialization:
	•	Computing random centers is fast but not great
	•	K-means++ is better as it picks point randomly then greedily adds the point further away 
	•	By default, K-means does 10 random restarts with different initialization
	•	For large datasets, K-means++ initialization may take much longer than clustering 

	•	Feature extraction using K-means:
	•	Add categorical features by saying it belongs to this cluster
	•	Add continuous features saying the distance to the cluster is that
	•	Examples: partition low-dimensional space or extract features from high-dimensional spaces for image patches

	•	Agglomerative clustering: collection of clustering algorithms that all build upon the same principles 
	•	Declares each point its own cluster and merges the two most similar clusters until some stopping criterion is reached 
	•	Stopping criterion: number of clusters left
	•	No predict method use fit_predict
	•	Agglomerative clustering produces hierarchical clustering 
	•	Detailed view
	•	Relies on 2D nature of data so can’t be used on datasets that have more than 2 features


	•	Dendrogram can visualize hierarchical clusters that can handle multidimensional datasets
	•	Helpful if you don’t have many data points

	•	Data points joined first are 1,4 2nd 6,9, etc
	•	Length of branch/bats shows how far apart the merged clusters are 

	•	There are linkage criteria that specify how most similar cluster is measured
	•	Single linkage: smallest minimum distance; like computing the minimum spanning tree then cutting the longest edge
	•	Ward (default choice): picks two clusters to merge such that the variance within all clusters increases least; leads to clusters that are relatively equally sized
	•	Average linkage: merges clusters that have the smallest average distance between all their points
	•	Complete linkage: merges two clusters that have the smallest maximum distance between their points.
	•	Use average or complete if one cluster is much bigger than all the others so dissimilar numbers of data points

	•	Pros and cons
	•	Can restrict to input topology graph so you can look at only merge things that are neighboring
	•	Retrictive topology can speed up the algorithm as you only consider few merges
	•	Fast with sparse connectivity otherwise O(log(n)n**2)
	•	Some linkage criteria can lead to very imbalanced cluster size (pro or con)
	•	Hierarchal clustering gives more holistic view and can help pick number of clusters

	•	DBSCAN doesn’t require the user to set the number of clusters a priori
	•	Capture clusters of complex shape
	•	Identify points that are not part of any cluster – can deal with outliers
	•	Slower but scales to relatively large datasets as there are 2 parameters to adjust; neighborhood radius (epsilon) and minimum number of samples 
	•	Identifies dense regions called core points
	•	boundary points: points that are within distance eps of core points
	•	Noise: points that don’t belong
	•	Less points than min_samples points within distance eps of starting point, point is classified as noise
	•	How it works:
	•	Clusters are formed by core samples 
	•	Sample is core sample if more than min_samples is within epsilon (dense region)
	•	Start with core sample and recursively walk neighbors that are core-samples and add to cluster
	•	Also add samples within epsilon that are bot core samples
	•	If can’t reach any more points, pick another core sample and start a new cluster
	•	If newly point has not enough neighbors, label outlier or noise
	•	Outliers can be later relabeled to belong to a cluster

	•	Boundary points may change depending on the order in which points are visited
	•	Not that important as we don’t usually have many boundary points
	•	Eps: defines cluster size
	•	Small eps no points are core samples, and all are noise
	•	Large eps: all points form 1 cluster – fewer but larger clusters
	•	min_samples: defines the outliers
	•	Increasing min_samples: fewer core points and more points labeled as noise
	•	Small min_samples: clusters form in area they wouldn’t, leading to more noise in the form of wrong clusters 
	•	While you don’t have to set the number of clusters explicitly, setting eps controls how many clusters will be found 
	•	Easier to find good setting for eps after scaling

	•	Pros and cons:
	•	Can learn arbitrary cluster shape and detect outliers
	•	Needs two parameters to adjust (eps hard to pick but can be done based on number of clusters)

	•	(Gaussian) Mixture models: unsupervised models that try to fit per metric density to your data
	•	Generative model – find p(x)
	•	
	•	Creates parametric density model (goal)
	•	How likely it is to observe a new point; can be used for outlier detection
	•	Clustering (goal)
	•	feature extraction
	•	Assumption:
	•	Data is a mixture of small number of known distributions
	•	Each mixture component distribution can be learned simply
	•	Each point comes from one component 
	•	Probability estimates:
	•	Probability of a point belonging to each of the components (predict_proba)
	•	How likely you observe X
	•	score(X[, y]): Compute the per-sample average log-likelihood of the given data X. 
	•	score_samples(X): Compute the weighted log probabilities for each sample.

	•	Summary:
	•	KMeans: Classic, simple. Only convex cluster shapes, determined by cluster centers, depends on initialization
	•	Agglomerative: Can take input topology into account, can produce hierarchy. 
	•	K-means and agglomerative allows you to specify number of clusters
	•	DBSCAN: Arbitrary cluster shapes, can detect outliers, often very different sized clusters.
	•	Gaussian Mixture Models (upgraded KMeans): Can model covariance, soft clustering sp how likely does it come from a particular component, can be hard to fit.

	•	Non-negative matrix factorization (NMF): latent representation and latent features are nonnegative

	•	Non-negative matrix factorization – unsupervised learning algorithm
	•	Like PCA it reduces dimension
	•	Instead of finding orthogonal components we find components and coefficients to be non-negative
	•	Matrix must be positive; each entry is positive 
	•	Can identify original components that make up the combined data
	•	More interpretable components than PCA 
	•	Advantages:
	•	data points are composed into positive sums
	•	Positive weights can be easier to interpret as each data point is represented as a positive combination of columns in W
	•	No cancellation like in PCA as you don’t have components in opposite directions
	•	Meaningful signs and no sign ambiguity
	•	0-1 data set 
	•	Expressed using PCA: not so clear due to cancellation effect
	•	Disadvantages:
	•	Can only be applied to non-negative data 
	•	Whether components are interpretable is a hit or miss; unsupervised model so outcome might be anything 
	•	Non-convex optimization, require initialization
	•	Not orthogonal
	•	Can be slow on large datasets because you look at all many possible variations of what can be seen as latent representation of your data points 
	•	changing number of components affects results (look below*)

	•	X=WH
	•	W: weights
	•	H: hidden representation
	•	*
	•	If you factor X into two components, each data point in the dataset can be represented as a positive combination of those two components
	•	If number of components = number of features in X, NMF can perfectly reconstruct the data and components will point towards the extreme points
	•	If number of components is 1, NMF finds best single direction to explain data and will point towards the mean
	•	Components are not orders by importance or variance and each component plays an equal role in explaining the data 
	•	NMF uses random initialization which leads to different results depending on the random seed
	•	Larger effect in complex situations

	•	Outlier detection:
	•	Find points that are different within the training set
	•	“Novelty detection”: no outlier in the training set
	•	In outlier detection your training dataset also contains outlier but novelty detection your data is clean
	•	Outliers are not labeled (otherwise imbalanced classification)
	•	Outlier and novelty detection ate often used interchangeably 
	•	Applications: 
	•	Fraud detection
	•	Network failure detection
	•	Intrusion detection
	•	Defect detection
	•	News? Intelligence?
	•	Basic idea:
	•	Model data distribution p(X)
	•	Outlier p(X)<e
	•	For outlier detection: be robust in modelling so robust to outliers
	•	Nonparametric or parametric (assume data follows a certain distribution) method
	•	Apply at test-time, threshold likelihood
	•	Task is generally ill-defined; so, you don’t know how the data looks like so you can’t detect which ones are outliers

	•	Leads to elliptic envelope outlier detection:
	•	Parametric method
	•	Assumes data follows gaussian distribution
	•	Fits a gaussian into data and look at data not fitted well by Gaussians. These points are outlier
	•	Fit robust covariance matrix and mean to identify the central location and shape of the ellipse 
	•	Compute Mahalanobis distance for each point and points with mahalanobis distance greater than a threshold are outliers
	•	Only works if gaussian assumption is reasonable
	•	Preprocessing with PCA might help sometimes

	•	Kernel density:
	•	Non-parametric method for estimating PDF of RV
	•	Estimates distribution based on data points provided
	•	Place a smooth bump at each data point by putting a small Gaussian blob around each data point and add all these bumps to form a smooth curves that represents pdf
	•	Doesn’t work well in high dimensions as data points are sparsely distributed so most of the space will be empty making it hard for kernels to overlap 
	•	In high dimensions you need to adjust your kernel bandwidth 
	•	Small bandwidth makes structure more detailed and less smooth - overfitting 
	•	Large bandwidth might create overly generalized densities that don’t reflect the data
	•	Its an unsupervised model so cross validation can be used to pick the bandwidth but can be skewed by outliers
	•	Addressing outlier and selecting bandwidth can improve performance

	•	One class SVM:
	•	Also uses gaussian kernel to cover data 
	•	Goal is to find a boundary in the feature space that encloses most of the data points
	•	So, it only selects support vectors (not all points) that lie closest to the boundary
	•	Specify outlier ratio (contamination) via nu (ν: fraction allowed to lie outside the boundary)
	•	Requires tuning parameter

	•	Isolation forests (also nonparametric):
	•	No parameters to tune
	•	Build random trees; If you must go very deep into the tree, then it’s not an outlier 
	•	Therefore, you measure the path length so how many splits (tree levels) are required to isolate a point 
	•	Outliers are easy to isolate from the rest as they tend to differ from most of the data 

	•	
	•	Robust covariance:
	•	Fits gaussian model to data assuming it forms a single gaussian distribution
	•	If data is a single isotropic gaussian it performs perfectly
	•	If data has multiple Gaussians it tries to merge them into 1 and fails to model the data correctly 
	•	If clusters are too far apart, it misrepresents the covariance (shape of spread)
	•	Isolation forest
	•	Find dense regions by isolating points using random splits with trees
	•	Works well across various cases because it simply focuses on separating dense areas from sparse
	•	Effective even in mixed datasets (gaussian + uniform noise)
	•	It doesn’t assume any specific data shape
	•	One-class SVM
	•	Uses a boundary to enclose most data points based on support vectors
	•	Works better with a clean training dataset where majority of points are normal
	•	Can misclassify points if there are outliers in training because boundaries are influenced by these points
	•	Using support vectors can lead to strange behavior if the data distribution is complex

	•	Summary:
	•	Isolation Forest works great!
	•	Density models are great if they are correct.
	•	Estimating bandwidth can be tricky in the unsupervised setting.
	•	Validation of results often requires manual inspection.
	•	One class SVM depends a lot on parameters

	•	Neural network models or MLPs
	•	Can be viewed as generalizations of linear models that perform multiple stages of processing to come to a decision
	•	In MLP, computing weighted sums is repeated multiple times, first computing hidden units which are combined using weighted sums to yield to result 
	•	So, it introduces intermediate steps called hidden layers, which allows the model to learn more complex relationships 
	•	Left is for logistic regression and right is for MLP
	•	 
	•	In the hidden layer, after multiplying each x with weight w, summing them and adding a bias, a non-linear function (tanh or ReLU) is applied to the sum
	•	We add the non-linear function cause otherwise it would be the same as logistic regression
	•	boundaries can now have different types of shapes 
	•	Linear models can’t capture complex pattern, but neural networks introduce non-linearity and is able to do so
	•	Relu: cuts off values below 0
	•	Tanh: saturates to -1 for low input values and +1 for high input values 
	•	
	•	Must set parameter for the number of nodes in the hidden layer 
	•	
	•	Hidden layers usually all have the same non-linear function, weights are different for each layer.
	•	Many layers → “deep learning”.
	•	For regression usually single output neuron with linear activation.
	•	For classification one-hot-encoding of classes, n_classes many output variables with softmax.

	•	Started as non-linear models for classification and regression
	•	Work well for very large datasets
	•	Require solving non-convex optimization so solution is dependent on initialization and method of optimization you choose
	•	Notoriously slow to train – need for GPU
